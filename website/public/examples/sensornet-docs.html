Overview

A lightweight binary protocol for IoT sensor devices to report telemetry data to a central hub.

  • All multi-byte integers use little-endian byte order.
  • Devices must increment sequence_num for each packet sent.
  • Hub responds to SENSOR_READING with ACK if status is abnormal.
  • Maximum packet size is 256 bytes.

import { SensorReadingEncoder, SensorReadingDecoder } from './generated/sensornet-protocol.js';

// Encoding (object → bytes)

const bytes = encoder.encode({

  "error": 0,
  "reserved": 0,

const decoded = decoder.decode();

Wire Format Example

Temperature sensor (type=0) reading 23.5°C with GPS location ID 42

Wire Format (Hex Bytes)
00000000 C0 0B 5C 61 00 00 BC 41 01 2A 00 00 00
Byte 0 (bits):
  • sensor_type = 0 (bits 0-3)
  • battery_low = 0 (bit 4)
  • error = 0 (bit 5)
  • reserved = 0 (bits 6-7)
Bytes 1-4: timestamp (uint32): 1633017600
Bytes 5-8: value (float32): 23.5
Byte 9: location present (uint8): yes
Bytes 10-13: location (uint32): 42
